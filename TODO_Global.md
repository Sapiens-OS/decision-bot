
# 📄 ТЗ для AI: Sapiens OS (Telegram MVP)

## **1. Общая идея проекта**

Sapiens OS — это система, которая помогает пользователю **принимать более осознанные решения**.

Это не просто чат-бот.
Это **Decision Assistant**, который:

* структурирует мышление пользователя
* помогает увидеть варианты и последствия
* сохраняет решения
* возвращается к ним позже для анализа результата

Долгосрочная цель:
создать систему, которая накапливает **данные о решениях и их последствиях**, и на основе этого улучшает качество рекомендаций.

---

## **2. Цель MVP**

Создать **Telegram-бота**, который:

1. Помогает пользователю разобрать решение
2. Сохраняет это решение в структурированном виде
3. Позволяет вернуться к прошлым решениям
4. Через время спрашивает результат решения (outcome tracking)

---

## **3. Основной пользовательский сценарий**

### Сценарий: создание решения

1. Пользователь вводит команду `/new`

2. Бот просит описать ситуацию

3. Бот задаёт уточняющие вопросы:

    * Что для вас важно в этой ситуации?
    * Какие варианты вы рассматриваете?
    * Что вас останавливает?

4. Бот генерирует структурированный ответ:

```
1. Ситуация
2. Варианты
3. Возможные последствия
4. Риски
5. Ключевой инсайт
```

5. Пользователь выбирает вариант (или пишет свой)

6. Всё сохраняется в базе

---

## **4. Вторичный сценарий: история**

Команда `/history`

Бот показывает:

* список решений
* краткое описание
* дату

При выборе решения:

* показывается полный анализ

---

## **5. Третий сценарий: follow-up (самый важный)**

Через время (например 7, 30, 90 дней):

Бот пишет:

```
Вы приняли решение: [название]

Как это сработало?
```

Варианты ответа:

* намного лучше
* немного лучше
* без изменений
* хуже
* намного хуже

Или свободный текст.

Это сохраняется как:

```
actual_outcome
```

## **6. Минимальная структура данных**

### Users

```
id
telegram_id
created_at
```

---

### Decisions

```
id

problem
context
options (JSON)
chosen_option
reason
expected_outcome
created_at
```

---

### Outcomes

```
id
decision_id
outcome_text
outcome_score (-2 to +2)
created_at
```

---

## **7. Архитектура системы**

### Компоненты:

1. Telegram Bot
2. Backend API (или встроенная логика)
3. LLM API (например OpenAI)
4. Database (PostgreSQL)

---

### Поток данных:

```
User → Telegram → Bot → LLM → Response → DB
```

---

## **8. Требования к AI (логике ответов)**

AI должен:

1. Не давать прямых указаний “сделай X”
2. Помогать думать, а не решать за пользователя
3. Давать структурированные ответы
4. Учитывать долгосрочные последствия
5. Выявлять возможные когнитивные и эмоциональные искажения

---

## **9. Формат ответа AI**

Ответ должен быть строго структурирован:

```
📌 Ситуация
...

⚖️ Варианты
1. ...
2. ...

📊 Последствия
Краткосрочные:
Долгосрочные:

⚠️ Риски
...

🧠 Инсайт
...

👉 Рекомендация
(мягкая, без давления)
```

---

## **10. Технические требования**

* Язык: Python
* Telegram API
* LLM API
* PostgreSQL

---

## **11. Ограничения MVP**

Не нужно реализовывать:

* сложные графы (Life Graph)
* социальные функции
* сложные модели ML
* мобильное приложение

---

## **12. Что важно в MVP**

1. Простота
2. Быстрый отклик
3. Понятный UX
4. Сохранение данных решений
5. Follow-up механизм

---

## **13. Ожидаемый результат**

Рабочий Telegram-бот, который:

* можно запустить локально
* позволяет пройти полный цикл:

  ```
  решение → анализ → сохранение → follow-up
  ```
* демонстрирует ценность продукта

---

## **14. Критерий успеха MVP**

* пользователь возвращается хотя бы 2–3 раза
* пользователь сохраняет решения
* пользователь отвечает на follow-up
